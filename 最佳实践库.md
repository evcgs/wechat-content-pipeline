## 最佳实践 2026-02-27：使用大纲中的详细配图描述 ⭐⭐⭐（最重要！）

### 实践描述
生成图片时，直接使用大纲文件中已有的详细配图描述，而不是自己随意创建提示词。

### 为什么好
- 大纲中的配图描述是结合文章内容编写的，不是随便编的
- 描述非常详细，包括类型、布局、内容结构、配色、风格
- 用这个描述生成的图片效果很好，用户确认"就是要这种效果"

### 怎么复制
1. 打开大纲文件或文章文件
2. 找到"配图建议"部分
3. 直接复制详细的配图描述
4. 用作提示词（不需要修改！）

### 示例
**来自：** `wisediag-content/02_内容生产/大纲/2026-02-26_大模型时代的技术主权_大纲搭建.md`

**封面图提示词：**
```
大模型时代的技术主权，半写实+科技感风格，左侧是五角大楼建筑（代表国家权力），右侧是Anthropic/OpenAI等科技企业的抽象logo集群（代表企业技术主权），中间是发光的通用大模型神经网络可视化，左右两侧通过数据流连接，横幅构图，宽高比16:9，专业信息图表设计，深蓝色 vs 亮紫色/青色配色，暖金色渐变连接
```

**用户反馈：** "就是要这种效果" ✅

## 最佳实践 2026-02-26：一张一张生成并确认 ⭐⭐（重要）

### 实践描述
生成图片时，一张一张生成，每生成一张就等用户确认，用户确认后再生成下一张。

### 为什么好
- 避免批量生成后发现效果不好，浪费时间和资源
- 可以根据用户反馈及时调整
- 确保每一张图片都符合要求

### 怎么复制
1. 先生成第1张图片
2. 发给用户确认
3. 用户确认后再生成第2张
4. 重复直到所有图片完成

## 最佳实践 2026-02-26：重要更新记录在 README.md 中 ⭐⭐（重要）

### 实践描述
每次重要更新后，在 README.md 中记录更新内容，让用户可以看到本次做了什么重要更新。

### 为什么好
- 用户可以快速了解最近的重要更新
- 不需要翻遍整个文档找更新内容
- 透明、可追溯

### 怎么复制
1. 在 README.md 开头添加"重要更新记录"部分
2. 记录每次的重要更新
3. 更新说明要是非常重要的使用说明

## 最佳实践 2026-02-26：每一步都把生成内容发到聊天界面 ⭐⭐（重要）

### 实践描述
每一步完成后，都把生成的内容直接发到聊天界面，而不是只发文件路径。

### 为什么好
- 用户无法点击文件路径查看
- 用户需要立即看到结果
- 透明、可追溯

### 怎么复制
1. 每一步完成后，保存文件到对应目录
2. 把生成后的内容直接发到聊天界面
   - 文字内容直接发
   - 图片文件直接发（用 message 工具）
3. 告诉用户文件已保存
