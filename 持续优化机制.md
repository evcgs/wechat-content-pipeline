## 一、每次完成文章后的回顾清单

### 回顾 1：问题记录

**必须记录：**
- [ ] 这次遇到了哪些问题？
- [ ] 问题的根本原因是什么？
- [ ] 是怎么解决的？
- [ ] 以后怎么预防？

**记录位置：** `问题记录库.md`

---

### 回顾 2：做得好的地方

**必须记录：**
- [ ] 这次哪些地方做得好？
- [ ] 为什么做得好？
- [ ] 怎么复制到下次？

**记录位置：** `最佳实践库.md`

---

### 回顾 3：工作流评估

**必须评估：**
- [ ] 当前的10步工作流是否合理？
- [ ] 是否需要增加/删除/调整步骤？
- [ ] 检查清单是否完整？

**记录位置：** `工作流迭代记录.md`

## 二、每周总结机制

### 每周日晚上做一次总结

**总结内容：**
1. **本周遇到的所有问题汇总**
   - 列出所有问题
   - 分类（图片生成、文字修改、发布等）
   - 分析趋势

2. **本周最佳实践汇总**
   - 列出所有做得好的地方
   - 提炼可复用的经验

3. **技能文档更新评估**
   - 是否需要更新技能文档？
   - 是否需要增加新的检查清单？
   - 是否需要调整警告的位置？

4. **工作流迭代评估**
   - 是否需要调整工作流步骤？
   - 是否需要增加新的环节？

## 四、优化决策树

### 发现问题后的决策流程

```
发现问题
    ↓
记录到「问题记录库」
    ↓
分析根本原因
    ↓
是工作流问题吗？
    ├─ 是 → 评估是否需要调整工作流
    │         ↓
    │      更新「工作流迭代记录」
    │         ↓
    │      更新技能文档（如果需要）
    │
    └─ 否 → 是检查清单问题吗？
              ├─ 是 → 更新检查清单
              │         ↓
              │      更新技能文档
              │
              └─ 否 → 是提示词问题吗？
                        ├─ 是 → 更新「提示词编写规范」
                        │         ↓
                        │      更新「提示词示例库」
                        │
                        └─ 否 → 记录到「问题记录库」，后续观察
```

## 六、优化的核心原则

### 原则 1：预防 > 治疗

- 与其事后返工，不如事前检查
- 把容易出错的地方做成检查清单
- 强制要求确认

---

### 原则 2：明确 > 模糊

- 把"应该做"变成"必须做"
- 用检查清单强制确认
- 用红色警告标注重要内容

---

### 原则 3：醒目 > 隐蔽

- 把重要的警告放在最醒目的位置
- 用大号字体、红色标注
- 新用户首先看到"最常见的坑"

---

### 原则 4：持续迭代 > 一次完美

- 不要追求一开始就完美
- 每次发现问题都迭代
- 每周都回顾和总结
- 持续优化，不断提升
