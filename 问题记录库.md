## 问题 2026-02-27：图片生成工作流缺陷 ⭐⭐⭐（严重）

### 问题描述
- 跳过了"结合文章内容二次生成详细配图描述"环节
- 自己随意创建简化版提示词
- 加入了过多技术约束（颜色代码、像素高度、栅格系统等）
- 导致图片效果差，需要大量返工

### 根本原因
1. 工作流中没有明确要求"必须使用已有的详细配图描述"
2. 没有强制检查机制
3. 技能文档中虽然提到了，但不够醒目，容易被忽略

### 解决方案
1. 在SKILL.md开头增加"重要警告"部分，用红色标注
2. 增加"图片生成前强制检查清单"
3. 明确禁止添加技术约束
4. 要求必须使用文章/大纲中已有的详细配图描述

### 预防措施
1. 图片生成前必须先确认检查清单
2. 必须使用已有的详细配图描述，不能自己编
3. 一张一张生成并确认
4. 定期回顾这个问题，避免重复踩坑

## 问题 2026-02-27：文字修改后没有联动检查 ⭐⭐（严重）

### 问题描述
- 文字修改后，没有检查哪些环节需要同步修改
- 文字修改后，没有同步修改图片的位置和详细描述
- 导致内容不一致

### 根本原因
1. 没有建立联动检查机制
2. 没有明确的检查清单

### 解决方案
1. 增加"文字修改后联动检查清单"
2. 明确每个步骤修改后需要检查的内容

### 预防措施
1. 修改任意步骤的文字后，必须先确认联动检查清单
2. 检查所有相关环节是否需要同步修改
3. 确保内容一致性

2. 在 TOOLS.md 文件中记录 API key
3. 在记忆文件中记录这个操作

1. API key 必须永久保存到 ~/.openclaw/.env 文件
2. API key 必须在 TOOLS.md 文件中记录
3. 所有重要操作必须在记忆文件中记录
