# 提示词编写规范

**日期：** 2026-02-28
**目的：** 统一图片生成提示词的编写规范，避免昨天遇到的问题

---

## 一、核心原则

### 原则 1：必须使用已有的详细配图描述 ⭐⭐⭐（最重要！）

**❌ 禁止：**
- 自己随意创建提示词
- 跳过"结合文章内容二次生成详细配图描述"环节
- 简化已有的详细描述

**✅ 必须：**
- 使用文章/大纲中已有的详细配图描述
- 那个描述是结合文章内容编写的，不是随便编的
- 如果是大纲阶段，就用大纲中的配图描述
- 如果是文章阶段，就用文章中的配图建议

---

### 原则 2：不要添加技术约束 ⭐⭐⭐（重要！）

**❌ 禁止添加：**
- ❌ 颜色代码（#0F172A、#1E40AF等）
- ❌ 像素高度（30像素）
- ❌ 技术术语（12列栅格、4px粗细等）
- ❌ "标题"这个词（AI会理解错，改成"文字"）
- ❌ 过度强调留白位置（让AI自由发挥）

**✅ 只保留：**
- ✅ 详细的画面内容描述
- ✅ 基本风格描述（如"现代极简主义+科技信息图表风格"）
- ✅ 宽高比（如"横幅构图，宽高比16:9"）

---

### 原则 3：一张一张生成并确认 ⭐⭐（重要！）

**❌ 禁止：**
- 批量生成多张图片
- 不等用户确认就继续生成

**✅ 必须：**
- 一张一张生成
- 每生成一张就等用户确认
- 用户确认后再生成下一张

---

## 二、不同类型图片的提示词模板

### 模板 1：封面图

**基础结构：**
```
[文章标题]，[风格]，[详细画面描述]，横幅构图，宽高比16:9，[补充风格说明]
```

**示例（来自实际项目）：**
```
大模型时代的技术主权，半写实+科技感风格，左侧是五角大楼建筑（代表国家权力），右侧是Anthropic/OpenAI等科技企业的抽象logo集群（代表企业技术主权），中间是发光的通用大模型神经网络可视化，左右两侧通过数据流连接，横幅构图，宽高比16:9，专业信息图表设计，深蓝色 vs 亮紫色/青色配色，暖金色渐变连接
```

---

### 模板 2：对比表/信息图表

**基础结构：**
```
[图表主题]，清晰的[布局]，[具体内容]，配色：[配色方案]，清晰无重影，专业信息图表风格，扁平化设计
```

**示例（来自实际项目）：**
```
两条路径的本质差异对比表，专业信息对比表格图表，两列对比布局，左侧列标题"传统工业/前沿技术路径（军用转民用）"，右侧列标题"当前大模型技术路径（民用转军用）"，对比维度：技术源头、控制权归属、扩散逻辑、迭代动力、安全逻辑，左侧用深蓝色系，右侧用亮紫色系，简洁专业的企业级信息图表，带清晰的标题和图标
```

---

### 模板 3：概念对比图

**基础结构：**
```
[主题]，左右分屏对比，中间用大号箭头标注"根本性反转"，左侧：[左侧内容]，右侧：[右侧内容]，配色：左侧冷色调（深蓝/灰），右侧暖色调（青/紫），现代扁平化设计，带简洁的视觉隐喻
```

---

### 模板 4：流程图/时间线

**基础结构：**
```
[主题]，水平时间轴，关键事件节点标注，[具体事件节点]，每个节点带图标和简要说明，关键节点用不同颜色高亮（红色=冲突点，绿色=妥协点），时间轴用深灰色，事件节点用彩色区分，专业新闻式时间线，清晰易读
```

## 三、提示词检查清单

在使用提示词生成图片之前，必须先确认：

- [ ] **我确认使用了文章/大纲中已有的详细配图描述**
- [ ] **我确认没有自己随意创建提示词**
- [ ] **我确认没有添加颜色代码、像素高度等技术约束**
- [ ] **我确认没有使用"标题"这个词（改成了"文字"）**
- [ ] **我确认会一张一张生成并确认**

**⚠️ 确认以上所有项都满足后，再开始生成图片！**

## 四、常见错误示例

### ❌ 错误示例 1：添加了太多技术约束

```
错误：
对比表，深蓝色背景(#0F172A)，30像素高度的标题，12列栅格布局，4px粗细的边框...

正确：
两条路径的本质差异对比表，专业信息对比表格图表，两列对比布局...
```

---

### ❌ 错误示例 2：自己随意创建简化提示词

```
错误：
封面图，科技感，左右对比...

正确：
大模型时代的技术主权，半写实+科技感风格，左侧是五角大楼建筑...
```

---

### ❌ 错误示例 3：使用"标题"这个词

```
错误：
标题：大模型时代的技术主权...

正确：
文字：大模型时代的技术主权...
```

## 五、最佳实践

### 实践 1：从大纲/文章中复制粘贴

**最佳做法：**
1. 打开大纲文件或文章文件
2. 找到"配图建议"部分
3. 直接复制详细的配图描述
4. 用作提示词（不需要修改！）

---

### 实践 2：一张一张确认

**最佳做法：**
1. 先生成第1张图片
2. 发给用户确认
3. 用户确认后再生成第2张
4. 重复直到所有图片完成

---

### 实践 3：记录好的提示词

**最佳做法：**
- 把效果好的提示词记录下来
- 建立"提示词示例库"
- 后续类似场景可以参考

---

## 六、总结

### 记住这3点：

1. **用已有的，不要自己编** ⭐⭐⭐
2. **去掉技术约束，只保留画面描述** ⭐⭐⭐
3. **一张一张确认** ⭐⭐
